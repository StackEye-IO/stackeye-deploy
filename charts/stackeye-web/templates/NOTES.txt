{{- if not .Values.sealedSecrets.enabled }}

⚠️  WARNING: You are using plain Kubernetes Secrets.
    For production, enable Sealed Secrets:

    1. Create a sealed secret:
       kubectl create secret generic {{ include "stackeye-web.fullname" . }}-secrets \
         --from-literal=AUTH0_SECRET='$(openssl rand -hex 32)' \
         --from-literal=AUTH0_CLIENT_SECRET='...' \
         --dry-run=client -o yaml | kubeseal --format yaml

    2. Update values.yaml:
       sealedSecrets:
         enabled: true
         encryptedData:
           AUTH0_SECRET: <encrypted-value>
           AUTH0_CLIENT_SECRET: <encrypted-value>
{{- end }}

NOTE: NEXT_PUBLIC_* environment variables are set at Docker image build time.
If you need different values, rebuild the image with different --build-arg values.
